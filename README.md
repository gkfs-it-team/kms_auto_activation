#### АВТОМАТИЧНА АКТИВАЦІЯ WINDOWS ТА OFFICE

Цей командний скрипт (run_mas_kms.cmd) призначений для автоматизації процесу 
активації операційної системи Windows та пакетів Microsoft Office (включаючи 
Office 2010 та сучасні Office 2016/2019/2021) за допомогою публічних 
KMS-серверів.

Після успішної активації скрипт застосовує відомий твік від спільноти MAS 
(Microsoft Activation Scripts), який підміняє адресу KMS-сервера в реєстрі 
на локальну заглушку (10.0.0.10). Це назавжди блокує появу настирливих 
повідомлень та банерів Microsoft про те, що "Ваша ліцензія не є автентичною".

-------------------------------------------------------------------------------

1. ОСНОВНІ ФУНКЦІЇ

-------------------------------------------------------------------------------

* Активація Windows: Інсталює універсальний GVLK-ключ та надсилає запит на 
  активацію до сервера kms8.msguides.com.
* Активація Microsoft Office 2010: Шукає встановлений Office 2010 (як 32-bit, 
  так і 64-bit), монтує відповідні KMS-ключі (Pro Plus / Standard) та 
  активує його.
* Конвертація та активація сучасних Office (2016/2019/2021): 
  - Якщо скрипт знаходить роздрібну версію (Retail), він автоматично конвертує 
    її у корпоративну (Volume) за допомогою системної утиліти 
    OfficeClickToRun.exe.
  - Після цього встановлює KMS-ключі та активує продукт.
* MAS Твік (leavenonexistentkms): Прописує в реєстр для Windows та Office 
  фейкову IP-адресу KMS-сервера (10.0.0.10)[cite: 6]. Активація при цьому 
  зберігається, але Office більше не може зв'язатися з серверами Microsoft 
  для перевірки валідності, що прибирає вотермарки та банери.
* Логування результатів: Усі кроки, помилки та статуси виконання записуються 
  у файл звіту C:\Windows\kms_report.txt.

-------------------------------------------------------------------------------

2. ЯК ЗАПУСТИТИ ВРУЧНУ

-------------------------------------------------------------------------------

Оскільки скрипт працює з системними службами активації (slmgr.vbs / ospp.vbs) 
та модифікує захищені гілки реєстру, йому ОБОВ'ЯЗКОВО потрібні права 
адміністратора.

1. Натисніть правою кнопкою миші на файл "run_mas_kms.cmd".
2. Виберіть "Запуск від імені адміністратора" (Run as administrator).
3. Зачекайте завершення роботи. У скрипті встановлено стартову затримку в 
   20 секунд для підготовки служб, тому консоль може деякий час залишатися 
   неактивною.

3. РОЗГОРТАННЯ ЧЕРЕЗ ГРУПОВІ ПОЛІТИКИ (GPO) В ДОМЕНІ

-------------------------------------------------------------------------------

Щоб автоматизувати активацію на великій кількості комп'ютерів у мережі Active 
Directory, найкраще використовувати Startup Scripts (Скрипти при запуску 
комп'ютера). Вони виконуються з найвищими привілеями системи 
(NT AUTHORITY\SYSTEM) ще до входу користувача в систему.

Крок 1. Перенесення файлу в SYSVOL
Скопіюйте файл run_mas_kms.cmd на ваш контролер домену у загальну папку 
скриптів.
Рекомендований шлях: \\yourdomain.local\SYSVOL\yourdomain.local\Scripts\

Крок 2. Створення об'єкта GPO

1. Відкрийте консоль керування груповими політиками (gpmc.msc).
2. Створіть новий об'єкт GPO з назвою, наприклад, Windows_Office_AutoActivation.
3. Зв'яжіть його з Організаційним підрозділом (OU), в якому знаходяться 
   КОМП'ЮТЕРИ (робочі станції), а не користувачі.

Крок 3. Призначення скрипту запуску

1. Натисніть правою кнопкою миші на створений GPO та оберіть Edit (Редагувати).
2. Перейдіть у розділ: 
   Конфігурація комп'ютера (Computer Configuration) 
   --> Політики (Policies) 
   --> Налаштування Windows (Windows Settings) 
   --> Скрипти (Scripts (Startup/Shutdown)).
3. Двічі клацніть на пункт "Запуск" (Startup).
4. У вікні натисніть кнопку "Додати..." (Add...).
5. У полі Script Name вкажіть повний мережевий шлях до файлу:
   \\yourdomain.local\SYSVOL\yourdomain.local\Scripts\run_mas_kms.cmd
6. Натисніть OK та застосуйте налаштування.

Крок 4. Важливий твік для запобігання зависань
Оскільки у скрипті використовуються команди timeout (паузи очікування відповіді 
серверів)[cite: 1, 4], за замовчуванням Windows може затримати відображення екрана 
входу для користувача. Щоб цього не сталося, увімкніть асинхронний запуск:

1. В цьому ж вікні редагування GPO перейдіть у: 
   Конфігурація комп'ютера --> Адміністративні шаблони --> Система --> Скрипти.
2. Знайдіть параметр "Виконувати скрипти запуску асинхронно" 
   (Run startup scripts asynchronously).
3. Переведіть його в стан Увімкнено (Enabled) та збережіть.

Тепер скрипт буде виконуватися у фоновому режимі, абсолютно непомітно для 
користувача. Політика відпрацює після наступного перезавантаження ПК. Для 
примусового тестування на клієнтській машині виконайте: "gpupdate /force" 
та перезавантажте систему.

5. ЗАСТЕРЕЖЕННЯ

-------------------------------------------------------------------------------

ВАЖЛИВО:

1. Інтернет: Для першого запуску скрипту комп'ютер обов'язково повинен мати 
   прямий доступ до інтернету, щоб зв'язатися з сервером kms8.msguides.com. 
   Після того, як ліцензія підтвердиться і пропишеться твік 10.0.0.10, 
   інтернет для підтримки активації більше не знадобиться.
2. Антивіруси / Windows Defender: Захисні системи можуть блокувати запуск 
   файлу або видаляти його, реагуючи на спробу змінити ліцензійні параметри 
   реєстру Windows (може визначатися як HackTool:BAT/KMS або подібне). 
   Перед розгортанням рекомендується додати файл скрипту до білого списку 
   вашого антивіруса чи політики Defender.
3. Ліцензія та відмова від відповідальності. Цей скрипт надається виключно в ознайомчих
   та освітніх цілях. Використовуючи його, ви берете на себе повну відповідальність
   за дотримання ліцензійних угод компанії Microsoft.
